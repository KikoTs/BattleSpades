## 1. Verified API contracts

### `enet.pyd` (`G:\AoSRevival\aceofspades_nonsteam\enet.pyd`)

- `enet.Host(None, peerCount=1, channelLimit=1, incomingBandwidth=0, outgoingBandwidth=0)` — client-mode host.
- `host.compress_with_range_coder()` — works; both server and client must call it.
- Constants: `EVENT_TYPE_CONNECT/DISCONNECT/RECEIVE/NONE`, `PACKET_FLAG_RELIABLE/UNRELIABLE_FRAGMENT/UNSEQUENCED/NO_ALLOCATE`.
- `peer.connect(addr, channels, data)` — the **`data` argument is `PROTOCOL_VERSION = 168`**. Our server logs the connect-data byte but does not currently enforce a version check; we should add `if data != 168: disconnect(ERROR_SERVER_OUT_OF_DATE=2)`.

### `shared.bytes.pyd`

```python
ByteWriter()                       # default-construct
  .write_byte(int)                # uint8
  .write_short(int)               # signed int16
  .write_int(int)                 # signed int32  (use values within [-2^31, 2^31-1])
  .write_uint64(int)
  .write_float(float)
  .write_pystring(str)            # length-prefixed
  .write_string(str, size)        # fixed-width
  .write_string_size(str, size)
  .write(bytes)
  .pad(n)
  .tell() / .rewind()

ByteReader(bytes)
  .read_byte() .read_short() .read_int() .read_uint64()
  .read_float() .read_pystring() .read_string(n) .read(n)
  .read_reader(n)                 # sub-reader
  .data_left() .skip_bytes(n) .seek(n) .tell() .rewind()
```

**Endianness: little-endian** (`0x42 0x1234 0xCAFEBABE → 42 34 12 be ba fe ca`). Methods are snake_case, **not** camelCase — our reversed `shared/bytes.pyx` may need to expose the snake_case API too if anything currently calls `writeByte` etc.

### `shared.lzf.pyd`

- `compress(bytes) → bytes` — **real LZF**. 64×`'A'` → 9 bytes (`014141e03300014141`).
- `decompress(bytes, outsize) → bytes` — real LZF inverse.
- **`decompress_client(bytes, outsize) → bytes`** — permissive: accepts both real LZF *and* the "fake chunking" format (0x1F + 32-byte chunk markers, last chunk = `len-1` byte + body). This is why our server's chunking-only output works on the client.

Our [server/util.py](../server/util.py) `lzf_compress` is the chunking-only flavour. It's intentional ("ENet's range coder does the actual compression"), and the client tolerates it. **No fix needed**, but the function name is misleading — it's not really LZF compression. Worth a renaming pass when we touch it next.

### `shared.packet.pyd`

- All 17 key classes verified present with `id`, `read`, `write`, `generate`.
- **119 unique packet IDs**, max id = 118. Protocol doc claims 127 — the gap is because some IDs are duplicated (e.g. `54` MapDataStart and `55` MapSyncStart serve overlapping purposes).
- **ID collision at 0**: `AddServer`, `ClockSync`, `item` all claim id=0. Receive-side disambiguation by context: gameplay packets are `ClockSync`; master-server registration is `AddServer`.
- Both constructor patterns work: `Cls(reader)` and `p = Cls(); p.read(reader)`. Our codebase uses both.
- **`p.generate()` returns a `ByteWriter`** (not bytes). Wrap with `bytes(...)` for wire output.
- **First byte of generated bytes is the packet ID** (verified for `ClockSync`, `SteamSessionTicket`, `ChatMessage`, `MapDataValidation`).
- **Strings are null-terminated for chat-style fields** — `ChatMessage.value = "hello world"` → wire bytes end with `…68 65 6c 6c 6f 20 77 6f 72 6c 64 00`. Length-prefixed paths exist via `write_pystring` for blob fields like the Steam ticket.

### `shared.constants.py`

- `PROTOCOL_VERSION = 168`
- `MASTER_VERSION = 32`
- `PACKET_COMPRESSION = 1`
- `MAX_PACKET_DECOMPRESSION_SIZE = 1_000_000`
- `SPADES_GAME_APP_ID = 480` (test app ID; real AoS Steam ID is 224540)
- **Team wire IDs: `TEAM_SPECTATOR=0, TEAM_NEUTRAL=1, TEAM1=2, TEAM2=3`**

### XOR-with-steam-ticket layer

- Not present in `shared.packet` — confirmed by name search (no `xor`/`crypt`/`cipher` symbols).
- Lives at the connection layer (server-side: our [server/connection.py](../server/connection.py) `decrypt`).
- Activated when `SteamSessionTicket(105)` arrives; subsequent inbound packets are XOR-decrypted with the ticket bytes cyclically.
- **Outbound is not encrypted** — only the receive direction. The bot doesn't need to encrypt.

## 2. Implications for the test harness

The harness plan in [TEST_HARNESS.md](TEST_HARNESS.md) is sound. Concrete adjustments:

1. Use `bytes(packet.generate())` to materialise wire bytes (`generate()` returns `ByteWriter`).
2. Connect with `peer.connect(enet.Address(b'127.0.0.1', 32887), 1, PROTOCOL_VERSION)`. Channels=1 to match server's `channelLimit=1`.
3. For inbound 0x31-prefixed packets, use `shared.lzf.decompress_client(payload[1:], MAX_PACKET_DECOMPRESSION_SIZE)`.
4. For outbound, mirror our server's pattern: `chr(0x30) + lzf_compress(bytes(writer))`. Don't encrypt.
5. ID collision at 0 → bot must use the inbound class registry (id → class), and special-case the `AddServer`/`ClockSync` distinction by source/state.

Bot wire layer is ~30 lines, sketch:

```python
import enet, shared.packet as P, shared.bytes as B, shared.lzf as L
from shared.constants import PROTOCOL_VERSION, MAX_PACKET_DECOMPRESSION_SIZE

def encode(packet, prefix=0x30):
    body = bytes(packet.generate())
    framed = lzf_chunk(body)            # match server util.lzf_compress (chunking only)
    return chr(prefix) + framed

def decode(data, steam_key=None):
    if data[0] == '\x31':
        body = L.decompress_client(data[1:], MAX_PACKET_DECOMPRESSION_SIZE)
    else:
        body = data[1:]
    if steam_key:
        body = ''.join(chr(ord(b) ^ steam_key[i % len(steam_key)]) for i, b in enumerate(body))
    pid = ord(body[0])
    cls = PACKET_CLASSES.get(pid)
    p = cls(); p.read(B.ByteReader(body[1:]))
    return p
```
